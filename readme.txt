Проект: Сокращенные или короткие ссылки.

Запуск:
Скачивание репозитория
docker-composer up -d


Фронтенд:
Реализуется пользовательский интерфейс взаимодействия с пользователем,
Сохранят ссылки в localStorage для того чтобы не нагружать бэк,
и не создавать дубли новых коротких ссылок, направленных на один ресурс.
Но разные пользователи, сославшиеся на один ресурс получат разные короткие ссылки.


Бэкенд:
После получения длинной ссылки, получаем ее hash(внутренняя функция питона).
Полученное число под модулем преобразуем в 62 систему счисления, где базой служат символы:
"0123456789QWERTYUIOPASDFGHJKLZXCVBNMqwertyuiopasdfghjklzxcvbnm"
Тем самым полученная строка имеет примерно 10 символов нужных нам.
Далее проверяем в бд, есть ли уже данных набор символов. Если есть,
то базовую строку случайно перемешиваем и получаем новый набор символов, основываясь на hash(длинная ссылка).ф
Повторяем пока не найдем свободный набор символов. Далее его отправляем пользователю.

На данный момент не реализована проверка на то, что строка является ссылкой через регулярные выражения.
